# Specs

Este diretório guarda as specs ativas do fluxo SDD.

## Estrutura

```text
specs/
└── <feature>/
    ├── spec.md
    ├── plan.md
    └── tasks.md
```

## Uso

- copie o template de `specs/_template/`;
- renomeie o diretório para a feature em kebab-case;
- preencha `spec.md`, `plan.md` e `tasks.md`;
- use esses arquivos como contrato para implementação.
